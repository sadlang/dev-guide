# المساهمة في دليل مطوّري لغة ص

## المبدأ
الدليل مرجعٌ حيّ يتبع الكود. أي وصفٍ هنا يجب أن **يطابق الكود الفعليّ** في
[`sadlang/s-programming-language`](https://github.com/sadlang/s-programming-language)؛
عند التعارض، الكود هو الحقيقة — صحّح الدليل.

## سير العمل (worktree + dev + PR)
يتبع نفس نمط المستودع الرئيسيّ:

```bash
# فرع معزول من dev في worktree
git worktree add ../dg-<مهمة> -b agent/dg-<مهمة> origin/dev   # أو main لهذا الريبو
cd ../dg-<مهمة>
# ... تعديل + معاينة ...
mdbook build                         # يجب أن يبني بلا أخطاء
git add -A && git commit -m "docs: ..."   # موقّع GPG
git push -u origin agent/dg-<مهمة>
gh pr create --base main --title "..." --body "..."
```

## قواعد الكتابة
- **العربية أولًا**، والمصطلحات التقنيّة بالإنجليزية بين قوسين عند أوّل ذكر.
- كل فصل يبدأ بـ**سطر هدف** («ماذا ستتعلّم») ويُنهى بـ**«اقرأ بعده»**.
- **روابط ملف:سطر** للكود الحقيقيّ حيثما أمكن (مسارات نسبيّة من جذر المستودع الرئيسيّ).
- المخطّطات بـ**Mermaid** (` ```mermaid `)، والكتل البرمجيّة موسومة باللغة (` ```sad `, ` ```cpp `, ` ```bash `).
- لا تُكرِّر مصدر الحقيقة — اربط إلى `language-truth/` بدل نسخه.

## البناء والتحقّق قبل PR
```bash
mdbook build           # 0 أخطاء/تحذيرات روابط
mdbook test            # إن وُجدت أمثلة قابلة للاختبار
```

## البنية
أضف الفصل إلى `src/SUMMARY.md` (شجرة الفهرس) ثم أنشئ ملفه تحت القسم المناسب.
